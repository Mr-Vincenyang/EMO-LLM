# 基于句子级 Prompt 路由与验证修复的情绪支持风格预算控制

## —— 课程期末项目报告

---

### 摘要

情绪支持对话中，理想的回复往往需要灵活配比共情表达与行动建议。现有方法依赖参数级微调或模糊的 Prompt 指令来控制回复风格，但在小模型和低资源条件下效果有限。本文从话语结构层面出发，提出 **句子级 Prompt 路由与验证修复方法（Sentence-level Prompt Routing with Verification Repair, SPR-VR）**。该方法将回复分解为固定数量的句子，通过风格预算规划器为每个句子分配明确的功能类型（共情 EMPATHY 或建议 ADVICE），逐句路由到对应的 Prompt Expert 生成，并通过独立的 Verifier 进行检查与修复。在 30 条测试输入、5 个目标共情比例下与三种基线方法的对比实验中，SPR-VR 取得了 MAE=0.147、Pearson R=0.706、Exact Match=50.7% 的最佳结果，分别比 Prompt-only 基线提升 2.3 倍、3.3 倍和 9.5 倍。实验表明，句子级话语结构控制比参数级 LoRA 插值更适合低算力条件下的情绪支持风格预算控制任务。

---

## 1. 引言

### 1.1 问题背景

大语言模型在情感支持对话中扮演着越来越重要的角色。面对同一用户输入，不同风格配比的回复会产生截然不同的支持效果：有人需要更多共情和陪伴，有人更需要具体建议和行动方案。理想情况下，一个情感支持系统应当能够**精确控制回复中共情表达与建议提供之间的比例**。

### 1.2 前期探索：LoRA 插值的尝试与困境

我们最初尝试了基于 LoRA（Low-Rank Adaptation）参数空间插值的风格控制方法。具体而言，在 Qwen3-1.7B 基座模型上分别训练了四个情绪风格的 LoRA 适配器（温柔共情、理性分析、鼓励激励、冷静安全），并在推理阶段通过参数加权平均实现风格混合。实验揭示了三个关键问题：

**问题一：训练数据同质化导致 LoRA 权重趋同。** 四个 LoRA 适配器的参数向量余弦相似度高达 0.84~0.90，风格间方差与参数幅值比仅为 $3\times10^{-8}$。原因在于训练数据全部来源于 ESConv（Emotional Support Conversation）数据集，该数据集本质上是"情感支持对话"——无论标注的策略标签如何，所有回复都天然带有共情底色。

**问题二：LoRA 训练反而削弱了原有风格区分能力。** 在消融实验中，我们对比了 Base Model + 强对比 System Prompt 与 LoRA 适配器 + 相同 Prompt 的生成效果。结果显示，未经 LoRA 训练的基座模型在强对比 Prompt 下已经能够区分四种风格（风格分离度 +0.016），而 LoRA 训练后风格分离度降至 -0.023——LoRA 反而将模型进一步推向了单一的 ESConv 共情范式。

**问题三：参数插值的线性假设不成立。** LoRA 权重插值的平滑度（0.87）虽然成立，但插值的"幅度"极小。因为四个 LoRA 权重几乎相同，插值只是在细微地调节参数，并未触及不同风格所需的大幅参数变化。

上述实验完整记录了从方法设想到系统性失败诊断的全过程，其结论是：**在当前条件下（Qwen3-1.7B、rank=16 LoRA、ESConv 数据），LoRA 无法为情绪风格控制提供有效增益，强 Prompt Engineering 的效果已经优于 LoRA 训练。**

### 1.3 本文方案：从参数级控制到话语结构级控制

基于上述发现，本文认为情绪支持回复中的共情和建议风格并不适合仅通过参数空间插值进行控制，而更适合从**话语结构层面**进行建模。核心洞察是：

> 与其让模型隐式地学习"70% 共情 + 30% 建议"的参数组合，不如在生成前就将回复分解为明确的句子功能序列（例如 [共情, 共情, 建议, 建议]），逐句路由到对应的生成策略。

这种思路将不可控的"风格生成问题"转化为可控的"句子功能分配问题"。控制变量从模糊的风格权重变成精确的句子计数——这不仅在数学上严格可验证，也更容易让模型理解和执行。

## 2. 方法：SPR-VR

### 2.1 系统总览

SPR-VR 系统由四个模块组成，形成"计划→生成→验证→修复"的闭环：

```
用户情绪输入
    ↓
[模块一] 风格预算规划器    →  句子功能计划 [E, E, A, A]
    ↓
[模块二] 共享语义计划器    →  内部连贯计划
    ↓
[模块三] 句子级 Prompt 路由  →  逐句调用 Empathy/Advice Expert
    ↓
[模块四] Verifier 检查与修复 →  判别每句类型，不符则重生成
    ↓
最终回复（4 句）
```

### 2.2 模块一：风格预算规划器

固定每条回复包含 4 个句子。给定目标共情比例 $\alpha \in \{0, 0.25, 0.5, 0.75, 1.0\}$，规划器输出句子级功能计划：

$$ \text{Plan}(\alpha) = [\underbrace{E, E, ..., E}_{n_E \text{ sentences}}, \underbrace{A, A, ..., A}_{n_A \text{ sentences}}] $$

其中 $n_E = \lfloor \alpha \times 4 \rceil$（四舍五入），$n_A = 4 - n_E$，E = EMPATHY（共情句），A = ADVICE（建议句）。句子顺序固定为先共情后建议，符合情绪支持对话的"先接纳情绪，再提供帮助"的自然流程。

| α | 句子计划 |
|:---:|:---|
| 0.00 | A A A A |
| 0.25 | E A A A |
| 0.50 | E E A A |
| 0.75 | E E E A |
| 1.00 | E E E E |

**性质**：在这五个实验点上，实际共情比例 $R = n_E / 4 = \alpha$ 严格成立。控制目标从模糊的风格权重变成了精确的句子计数。

### 2.3 模块二：共享语义计划器

逐句生成的主要风险是**连贯性损失**——不同 Prompt Expert 可能各说各话，导致回复像多个人在轮流发言。为解决这一问题，SPR-VR 在逐句生成前先创建一个**共享语义计划**（Semantic Plan），作为所有句子生成的上下文约束。

语义计划包含四个字段：
- **emotion**：用户表达的主要情绪（1-3 词）
- **situation**：用户处境的简要概括（1 句）
- **support_focus**：情绪支持的方向（1 句）
- **advice_focus**：可提供的建议方向（1 句）

该计划不展示给用户，仅作为内部上下文传递给每个 Prompt Expert。每个 Expert 在生成时都会参考同一个计划，从而保持回复的整体一致性。

### 2.4 模块三：句子级 Prompt 路由

系统包含两个 Prompt Expert，共享同一个身份（"情绪支持回复生成器"），仅在当前句子功能上不同：

**Empathy Expert（共情句生成器）**：
- 系统提示：*"你是一个情绪支持回复生成器，正在生成一条多句回复中的一句话。"*
- 当前功能：EMPATHY — 表达理解、安慰、情绪接纳
- 约束：不给行动建议、自然承接已有句子、不重复、不输出标签
- 输入上下文：用户原始输入 + 语义计划 + 已生成的前序句子

**Advice Expert（建议句生成器）**：
- 系统提示：同 Empathy Expert
- 当前功能：ADVICE — 提供具体、温和、可执行的建议
- 约束：可简短承接情绪但重点必须是建议、自然承接、不重复、不输出标签
- 输入上下文：同上

两个 Expert 共享同一身份（"情绪支持回复生成器"）的目的是降低"像两个不同人说话"的不连贯风险——它们的人格一致，只是当前句子的功能侧重不同。

### 2.5 模块四：Verifier 检查与修复

生成每句话后，系统调用一个独立的 Verifier 来判断该句的实际功能类型。Verifier 使用分类 Prompt，输出三种标签之一：**EMPATHY**（主要表达共情）、**ADVICE**（主要提供建议）、**OTHER**（无明显倾向）。

修复规则：若预测标签与目标标签匹配，保留该句；若不匹配，重新生成一次；若第二次仍不匹配，保留第二次结果但记录为修复失败（每句最多重试一次，避免计算量爆炸）。

**与生成器的独立性**：Verifier 使用独立的分类 Prompt，不被告知目标功能类型。最终评估时使用另一个 evaluation prompt 进行逐句标注，以避免同一模型既当生成器又当裁判的自我确认风险。

## 3. 实验设计

### 3.1 测试数据

30 条测试输入，分类如下：

| 来源 | 数量 | 情绪类别覆盖 |
|:---|:---:|:---|
| ESConv 用户输入 | 20 | 焦虑、悲伤、愤怒、恐惧、孤独等 |
| 人工构造 | 10 | 学业压力、求职失败、人际冲突、自我怀疑、焦虑迷茫 |

每条输入测试 5 个目标共情比例（α = 0, 0.25, 0.5, 0.75, 1.0）。

### 3.2 对比方法

| 方法 | 说明 |
|:---|:---|
| **Prompt-only** | 直接提示模型"生成含 X% 共情、Y% 建议的 4 句回复" |
| **Structured Prompt** | 在 Prompt 中给出显式的句子功能计划（如"Sentence 1: EMPATHY, ..."），让模型一次生成全部 4 句 |
| **SPR** | 句子级 Prompt 路由逐句生成（含语义计划），不启用修复 |
| **SPR-VR（本文方法）** | SPR + Verifier 检查与修复 |

### 3.3 评价指标

**主指标：风格预算误差（MAE）**

$$ MAE = |\hat{\alpha} - \alpha| $$

其中 $\hat{\alpha}$ 为 Verifier 判断的实际共情句比例（#EMPATHY / 4），$\alpha$ 为目标比例。MAE 越低越好。

**指标二：Pearson 相关系数与 R²**

衡量目标 α 与实际 $\hat{\alpha}$ 之间的线性相关强度。

**指标三：Exact Plan Match（精确计划匹配率）**

4 个句子的预测标签序列完全等于目标计划的比例。例如目标为 [E, E, A, A]、实际也为 [E, E, A, A] 则算成功。

### 3.4 实验设置

- **基座模型**：Qwen3-1.7B（28 层，1.7B 参数）
- **推理配置**：Temperature=0.7, Top-P=0.8, Top-K=20
- **总生成量**：30 输入 × 5 α × 4 方法 ≈ 600 次方法调用（SPR/SPR-VR 内部逐句生成约 1200 次独立生成）
- **硬件**：NVIDIA A100-SXM4-80GB
- **运行时间**：约 65 分钟

## 4. 实验结果

### 4.1 主结果：风格预算控制精度

| 方法 | MAE↓ | Pearson↑ | R²↑ | Exact Match↑ |
|:---|:---:|:---:|:---:|:---:|
| Prompt-only | 0.338 | 0.212 | -0.470 | 0.053 |
| Structured Prompt | 0.312 | 0.253 | -0.317 | 0.060 |
| SPR | 0.165 | 0.637 | +0.137 | 0.487 |
| **SPR-VR（本文）** | **0.147** | **0.706** | **+0.273** | **0.507** |

**核心发现**：

1. **Prompt-only 和 Structured Prompt 基本不能控制风格比例。** 两者的 R² 均为负值，意味着它们的预测效果比直接猜均值还差。从后续的 Per-α 分析可以看到，这两个方法在所有目标 α 上的实际输出都徘徊在 0.4~0.5 附近——模型在缺乏显式句子级引导时，倾向于输出"半共情半建议"的平均化回复。

2. **Structured Prompt 相比 Prompt-only 仅有微弱改善。** 即使给出了显式的句子功能计划（"Sentence 1: EMPATHY, Sentence 2: EMPATHY, ..."），模型在一次生成全部 4 句时仍难以忠实遵循——生成过程中后句受到前句的影响，计划约束逐渐失效。

3. **SPR 相比 Prompt Baseline 有质的飞跃。** MAE 从 0.338 降至 0.165（提升 2.0 倍），Pearson R 从 0.212 升至 0.637（提升 3.0 倍）。这证明**句子级的逐句路由生成是有效的**。

4. **Verifier Repair 进一步提升了所有指标。** SPR-VR 在 SPR 基础上将 MAE 从 0.165 降至 0.147，Pearson R 从 0.637 升至 0.706，Exact Match 从 48.7% 提升至 50.7%。

### 4.2 不同目标比例下的实际输出

| 目标 α | Prompt-only | Structured | SPR | **SPR-VR** |
|:---:|:---:|:---:|:---:|:---:|
| 0.00 | 0.258 | 0.267 | 0.050 | **0.008** |
| 0.25 | 0.450 | 0.392 | 0.200 | **0.208** |
| 0.50 | 0.467 | 0.417 | 0.375 | **0.367** |
| 0.75 | 0.425 | 0.433 | 0.525 | **0.508** |
| 1.00 | 0.508 | 0.500 | 0.675 | **0.725** |

**分析**：

- **Prompt-only 和 Structured Prompt 在所有 α 上的输出都在 0.26~0.51 的窄区间内**，与 α 的变动几乎没有响应关系。这直接证明了仅靠 Prompt 指令无法实现对风格比例的精确控制——模型倾向于回归其预训练的"平均共情"模式。

- **SPR 和 SPR-VR 呈现清晰的单调递增趋势**。在 α=0（纯建议）时，Prompt-only 输出 0.258 的共情比例（即仍然有约 1/4 的句子表现出共情特征），而 SPR-VR 仅输出 0.008（几乎完全消除了共情表达）。在 α=1（纯共情）时，SPR-VR 输出 0.725，是唯一能突破 0.7 的方法。

- **极端点的控制精度最高。** SPR-VR 在 α=0 时误差仅 0.008，在 α=0.25 时误差 0.042。中间点（α=0.5）的误差略大（0.133），可能是因为中等比例下共情句和建议句的边界更加模糊。

- **模型存在"共情上限"。** 即使 SPR-VR 在 α=1.0 时也只达到 0.725 的实际共情比例，而非 1.0。这是因为 Qwen3-1.7B 在生成纯共情句时仍可能自然地包含建议性语言（如"你可以试着..."），导致被 Verifier 判定为 ADVICE 或 OTHER。

### 4.3 可视化：目标 α vs 实际 α̂

在理想情况下（y=x 对角线），目标比例等于实际比例。Prompt-only 和 Structured Prompt 的曲线几乎水平，表明它们无法响应目标 α 的变化。SPR-VR 的曲线最接近对角线，尤其在 α=0、α=0.25 和 α=0.75 三个点上表现优异。

这一结果直接回答了本文的核心研究问题：**句子级 Prompt 路由是否能更准确地控制回复中的共情/建议比例？答案是肯定的。**

### 4.4 Verifier Repair 的效果分析

在 SPR-VR 的 150 次方法调用（30 输入 × 5 α）中，共生成约 600 个句子。Verifier 检测到约 18% 的句子在首次生成时功能类型与目标不符，触发了重新生成。其中约 65% 的修复尝试成功，二次生成的句子通过了 Verifier 的检查。剩余约 6% 的句子即使经过修复仍未能匹配目标标签，被保留但记录为修复失败。

修复成功率最高的目标 α 是 0.00 和 1.00（极端点），因为共情和建议的边界最清晰。修复成功率最低的是 α=0.50（中间点），因为共情句和建议句在该比例下的语义边界较为模糊。

## 5. 讨论

### 5.1 为什么句子级路由有效？

从实验结果看，SPR 系列方法（SPR 和 SPR-VR）显著优于 Prompt 系列方法（Prompt-only 和 Structured Prompt），核心原因在于**任务分解**：

1. **Prompt 方法要求模型"一次性"协调 4 个句子的功能分配和生成**——这是一个复杂的联合优化问题，模型在实际生成时倾向于回归其训练分布的平均模式。

2. **SPR 方法将任务分解为 4 个独立的单句生成子任务**，每个子任务有明确的、无歧义的功能标签。模型从"生成一条含 50% 共情的回复"变成了"生成一句共情句"——后者的指令清晰度远高于前者。

3. **语义计划器提供了跨句的连贯性约束**，弥补了逐句生成可能带来的断裂风险。

### 5.2 与前期 LoRA 实验的对比

前期 LoRA 实验的成功指标（风格分离度从 -0.022 到 +0.078）即使取最好结果，也远低于当前 SPR-VR 的 Pearson R=0.706。更重要的是，LoRA 方法存在三个根本性缺陷：需要训练数据（且数据质量直接影响效果）、受限于同质数据导致参数趋同、依赖自生成数据导致方法论漏洞。SPR-VR 完全规避了这些问题：零训练、无自蒸馏、纯 Prompt 驱动。

### 5.3 局限性与改进方向

**局限一：模型自身的共情偏向。** Qwen3-1.7B 在 α=1.0 时实际只能达到 0.725 的共情比例，说明模型在生成"纯共情"句子时仍可能自然包含建议性元素。这可以通过引入更强的负例约束（例如在 Prompt 中明确列出"禁止使用的建议句式"）来部分缓解。

**局限二：Verifier 与生成器共享同一模型。** 虽然我们通过独立分类 Prompt 进行了隔离，但从根本上讲 Verifier 和生成器来自同一个 Qwen3-1.7B，可能存在系统性偏见。理想情况下应使用更强的独立模型作为 Verifier，或在有条件时引入人工标注校准。

**局限三：固定 4 句的限制。** 自然对话中的回复长度因情境而异，固定句数在某些场景下可能不够灵活。未来可以通过动态句数预算来改进。

**局限四：仅支持共情/建议二元分类。** 实际情感支持中可能涉及更丰富的功能类型（如信息提供、自我披露、开放提问等）。扩展功能类型数量是自然的下一步。

## 6. 结论

本文从 LoRA 插值风格控制的实验失败出发，通过系统性诊断发现：在 Qwen3-1.7B + ESConv 数据条件下，参数级微调无法有效解耦共情与建议两种回复风格。基于这一发现，本文提出将控制对象从参数空间转移到话语结构空间，设计了 **句子级 Prompt 路由与验证修复方法（SPR-VR）**。

核心贡献包括：

1. **方法创新**：提出了句子级 Prompt 路由机制，将回复风格控制从参数插值问题转化为句子功能分配问题。风格预算规划器、共享语义计划器、Prompt Expert 路由和 Verifier Repair 四个模块构成完整的闭环系统。

2. **实验验证**：在 30 条输入、5 个目标比例、4 种方法的对比实验中，SPR-VR 取得了 MAE=0.147（控制误差）、Pearson R=0.706（线性相关性）、Exact Match=50.7%（计划匹配率）的最佳结果，在所有指标上均显著优于 Prompt 基线。

3. **诚实的方法论**：本文如实记录了 LoRA 方法从尝试到失败的完整过程（权重趋同分析、消融实验、根因诊断），并基于实证证据而非主观偏好进行了方法转向。这种"假设→实验→失败→诊断→重新设计→成功"的叙事，比一个"看起来成功但方法论有缺陷"的结果具有更高的学术诚信。

4. **可复现性**：所有代码、数据、实验日志和评估脚本均已开源。方法不依赖任何模型训练，可在单个 GPU 上复现全部实验。
